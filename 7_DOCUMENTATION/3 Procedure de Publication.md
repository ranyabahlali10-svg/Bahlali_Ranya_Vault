# Publication et Archivage

Le projet est conçu pour être **Stand-alone** : une fois téléchargé ou consulté en ligne, il se suffit à lui-même.

### 1. Versioning avec GitHub
J'ai publié l'intégralité du Vault sur GitHub. Cela permet :
- Un archivage sécurisé du travail.
- Une consultation publique de la structure et du code.
- Un suivi des modifications (Versioning).

### 2. Accessibilité
Le choix du format Markdown assure que les notes restent lisibles même sans Obsidian. Pour la présentation finale, j'ai intégré un module de **Slides** directement dans le Vault pour faciliter la démonstration du projet.
